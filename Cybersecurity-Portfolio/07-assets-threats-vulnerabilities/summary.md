
# Assets, Threats, and Vulnerabilities — Summary

## What this section demonstrates
- Building an asset inventory and classifying sensitivity based on business impact
- Performing qualitative risk scoring using likelihood × impact
- Prioritizing risks to guide security resource allocation
- Applying least privilege to prevent data leaks and reduce blast radius

## Key takeaways
Security work often starts with understanding what must be protected (assets), what can go wrong (threats), and where weaknesses exist (vulnerabilities). Clear documentation, consistent scoring, and practical control recommendations support decision-making and improve organizational security posture.
