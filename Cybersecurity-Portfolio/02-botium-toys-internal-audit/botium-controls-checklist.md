# Controls and Compliance Checklist – Botium Toys (Practice)

## Controls Assessment

| Control                                   | In Place? | Notes                                                |
|-------------------------------------------|----------:|------------------------------------------------------|
| Least privilege                           | No        | All employees can access internal data.              |
| Disaster recovery plans                   | No        | No documented DR plans or tested procedures.        |
| Password policies                         | Partially | Policy exists but does not meet current standards.  |
| Separation of duties                      | No        | Not implemented for critical processes.             |
| Firewall                                  | Yes       | Rules configured and managed by IT.                 |
| Intrusion detection system (IDS)          | No        | Not currently deployed.                             |
| Backups                                   | No        | Critical data not backed up.                        |
| Antivirus software                        | Yes       | Installed and monitored.                            |
| Manual monitoring of legacy systems       | Yes       | Performed, but without a clear schedule/process.    |
| Encryption for sensitive data             | No        | Payment data stored unencrypted.                    |
| Password management system                | No        | Resets handled manually by IT.                      |
| Physical locks (office/store/warehouse)   | Yes       | Described as sufficient.                            |
| CCTV surveillance                         | Yes       | Up to date and functioning.                         |
| Fire detection/prevention                 | Yes       | Alarm and sprinkler systems in place.               |

## Compliance Best Practices (Summary)

**PCI DSS**
- Restricted access to cardholder data: **No**
- Secure storage and processing of card data: **No**
- Encryption of payment data: **No**
- Strong password management: **No**

**GDPR**
- EU customer data kept private/secured: **Partially**
- 72-hour breach notification plan: **Yes**
- Data classification and inventory: **Needs improvement**

**SOC (Type 1/2 – Internal View)**
- User access policies documented: **No**
- PII/SPII confidentiality enforced: **Partially**
- Data integrity and availability processes: **Partially**
